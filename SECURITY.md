# Security Policy

## Supported versions

Security fixes are considered only for the current `main` branch.

| Target | Supported |
| --- | --- |
| Current `main` | Yes |
| Historical commits, forks, and downstream modifications | No |

## Reporting a vulnerability

Please report suspected security vulnerabilities through GitHub's
[private vulnerability reporting form](https://github.com/dariero/VerdigrisE/security/advisories/new).

Do not include vulnerability details in a public issue, pull request, or discussion.
Include the affected commit, prerequisites, reproduction steps or a proof of concept,
expected impact, and any suggested mitigation you have. Remove credentials and
unnecessary personal data from the report.

Use [public issues](https://github.com/dariero/VerdigrisE/issues) for non-security bugs
and feature requests.

This policy does not establish response or remediation deadlines.
